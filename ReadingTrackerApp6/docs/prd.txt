0. 전제 사항 (Assumptions)

- 본 프로젝트는 **React Native** 기반의 모바일 앱(주요 타겟: iOS, Android)으로 개발한다.
- UI/UX는 모바일 환경에 최적화하며, 네이티브 앱의 성능과 일관된 사용자 경험을 목표로 한다.
- 데이터 저장은 초기에는 로컬(예: SQLite, AsyncStorage 등)로 구현하고, 추후 클라우드 동기화 기능을 확장할 수 있다.
- 앱 배포는 App Store, Google Play를 기본으로 한다.

0-1. 라이브러리 사용 제한 (Constraints on Libraries)

- UI 컴포넌트:
  - React Native 기본 컴포넌트(예: View, Text, ScrollView 등)와 React Native Paper, React Native Elements 등 범용 UI 라이브러리만 사용 가능
  - 웹 전용 UI 라이브러리(예: Material UI, Ant Design, Shadcn 등)는 사용 불가
- 네비게이션:
  - React Navigation만 사용 가능
- 상태 관리:
  - React Context, Redux Toolkit, Zustand 등 경량 상태 관리 라이브러리만 사용 가능
- 스토리지/DB:
  - AsyncStorage, SQLite, WatermelonDB 등 모바일 친화적 로컬 DB만 사용 가능
  - 브라우저 전용(LocalStorage 등) 사용 불가
- 차트/그래프:
  - react-native-svg 기반 차트 라이브러리(예: Victory Native, react-native-chart-kit 등)만 사용 가능
- 이미지/OCR:
  - expo-image-picker, react-native-camera, expo-mlkit-ocr 등 Expo/React Native 공식 지원 라이브러리만 사용 가능
- 기타:
  - Expo Managed Workflow를 우선 적용하며, Expo에서 지원하지 않는 네이티브 모듈은 사전 검토 후 도입
  - 라이선스가 불분명하거나, 유지보수가 중단된 라이브러리는 사용 금지

Product Requirements Document (PRD): 리브노트(Livenote)

문서 버전: 3.0 (Vision & Platform Roadmap)
작성자: AI Assistant
최종 수정일: 2023-11-02

1. 개요 (Overview)

1.1. Executive Summary (요약)
'리브노트(Livenote)'는 개인의 독서 경험을 기록하고 연결하여 지적 성장으로 이끄는 지능형 독서 플랫폼을 지향한다. 초기에는 React Native 기반 모바일 앱으로 시작하여 손쉬운 기록 경험을 제공하고, 점진적으로 AI 기반 분석, 시각적 사고 도구, 전문가 콘텐츠 생태계를 도입하여 '생각하는 사람들을 위한 필수 지식 동반자'로 성장하는 것을 목표로 한다.

핵심 가치는 "기록에서 발견으로, 발견에서 성장으로(From Logging to Discovery, From Discovery to Growth)"이다.

1.2. 문제 정의 (Problem Statement)
* 기록의 번거로움: 수동 기록은 꾸준함을 방해한다.
* 정보의 파편화: 기록된 지식들이 서로 연결되지 못하고 흩어져 있다.
* 성장의 불확실성: 자신의 지적 여정을 객관적으로 파악하고 다음 단계로 나아갈 방향을 찾기 어렵다.
* 지식 탐색의 비효율성: 신뢰할 수 있는 정보와 깊이 있는 인사이트를 얻는 데 많은 시간과 노력이 소요된다.

1.3. 목표 및 성공 지표 (Goals & Success Metrics)
* 궁극적 목표 (Ultimate Goal): 사용자가 읽고, 생각하고, 연결하는 모든 과정을 지원하여, 지식 탐구의 전 과정에서 가장 신뢰받는 플랫폼이 된다.
* 단계별 목표 및 지표: 각 마일스톤에서 별도 정의 (하단 '개발 로드맵 및 마일스톤' 참고)

2. 사용자 (Users)

* 핵심 사용자 (Core Users): (기존과 동일)
* 확장 사용자 (Expanded Users, Phase 4 이후): 특정 분야의 전문가, 교육자, 콘텐츠 크리에이터, 지식 콘텐츠 소비자.

3. 기능 요구사항 (Functional Requirements - 전체 목록)

Part 1: 핵심 기록 및 관리 (Core Logging & Management)
* F-01: API 기반 책 검색 및 등록
* F-02: 수동 책 등록
* F-03: 독서 상태 관리 (읽고 싶은/읽는 중/다 읽은)
* F-04: 인용문 기록 (OCR)
* F-05: 감상 메모 기록
* F-06: 독서 시간 기록 (타이머 + 수동 입력)
* F-07: 통합 보기 화면 (내 서재)
* F-08: 텍스트 검색
* F-12: 데이터 관리 (초기: 단순 백업, 확장: 클라우드 동기화)
* F-13: UI/UX (라이트/다크 모드)
* F-14: 사용자 온보딩
* F-15: 태그 관리 및 필터링

Part 2: 성장 및 동기부여 (Growth & Motivation)
* F-09: 통계 대시보드
* F-10: 핵심 지표 시각화 (차트)
* F-11: 기록 공유 (이미지 카드)
* F-17: 독서 목표 설정 및 관리 (챌린지)
* F-18: 연간 독서 회고 리포트

Part 3: 지식의 연결과 확장 (Connecting & Expanding Knowledge)
* F-16: 노트 간 연결 (위키 스타일 [[링크]])
* F-19: 인기 인용문 표시 (소셜 하이라이트)
* F-22: 지식 맵 시각화 (그래프 뷰)
* F-23: 자유형 아이디어 보드

Part 4: AI 지능 및 플랫폼 (AI Intelligence & Platform)
* F-20: AI 기반 인사이트 추출 (사용자 노트 분석)
* F-21: 대화형 책 탐색 (AI 북챗)
* F-24: 전문가 북큐레이션 구독
* F-25: 프리미엄 노트 마켓플레이스

4. 개발 로드맵 및 마일스톤 (Roadmap & Milestones)

개발 로드맵을 현실성과 임팩트를 고려하여 5단계의 명확한 마일스톤으로 재구성합니다.

마일스톤 1: 탄탄한 기반 다지기 (The Foundation)
* 기간: M0 – M3
* 테마: "가장 쉽고 아름다운 개인 독서 기록 도구"
* 주요 기능:
    * 핵심: F-01, F-02, F-03, F-04(OCR), F-05, F-07, F-14(온보딩)
    * 기술 기반: 로컬 DB(SQLite/WatermelonDB) 구조 완성
* 목표: 사용자가 아무런 마찰 없이 책을 등록하고 기록하는 핵심 경험을 완성한다.
* 성공 지표: 신규 가입자 수, 기록 생성 완료율 > 95%, 스토어 평점 ≥ 4.5

마일스톤 2: 습관 형성 및 동기부여 (Habit & Motivation)
* 기간: M3 – M6
* 테마: "숫자로 증명되는 나의 성장"
* 주요 기능:
    * 핵심: F-06(시간 기록), F-08(검색), F-09/F-10(통계/차트), F-11(공유), F-13(다크모드), F-17(독서 목표)
    * 기술 기반: 데이터 집계 및 시각화 라이브러리 도입
* 목표: 사용자가 자신의 성장을 눈으로 확인하고, 독서 습관을 유지할 강력한 동기를 부여한다.
* 성공 지표: Day-30 리텐션 ≥ 30%, WAU 당 평균 기록 생성 수 ≥ 1.5, 통계 화면 주간 조회 수.

마일스톤 3: 지식의 연결과 안정성 확보 (Connection & Stability)
* 기간: M6 – M10
* 테마: "흩어진 생각을 연결하고, 어디서든 접근 가능하게"
* 주요 기능:
    * 핵심: F-12(클라우드 실시간 동기화), F-15(태그 시스템), F-16(노트 간 연결)
    * 기술 기반: 서버 DB 및 API 구축, 사용자 계정 시스템 도입, 동기화 엔진 개발
* 목표: 여러 기기 사용자에게 완벽한 사용 경험을 제공하고, 태그와 링크를 통해 지식을 유기적으로 연결하는 기반을 마련한다.
* 성공 지표: 유료 전환율(동기화 기능 유료화 시), 다중 기기 사용률, 노트 당 평균 태그/링크플릭스 수.

마일스톤 4: 지능형 인사이트 제공 (Intelligent Insight)
*   기간: M10 – M15
*   테마: "나보다 나를 더 잘 아는 지식 비서"
*   주요 기능:
    *   핵심: F-18(연간 회고 리포트), F-19(소셜 하이라이트), F-22(지식 맵), F-20(AI 인사이트 추출)
    *   기술 기반: AI/ML 모델(임베딩, 토픽 모델링) 도입, 그래프 데이터베이스 또는 라이브러리 연구
*   목표: 단순 기록을 넘어, 축적된 데이터를 분석하고 시각화하여 사용자에게 새로운 발견과 인사이트를 제공한다.
*   성공 지표: "인사이트" 기능 만족도 설문, 지식 맵기능 사용 시간, 리포트 공유 횟수.

마일스톤 5: 플랫폼 생태계 구축 (Platform & Ecosystem)
* 기간: M15+
* 테마: "함께 성장하는 지식 커뮤니티"
* 주요 기능:
    * 핵심: F-21(AI 북챗), F-23(아이디어 보드), F-24(큐레이션 구독), F-25(마켓플레이스)
    * 기술 기반: 콘텐츠 저작자(Creator)를 위한 관리 도구, 결제 및 정산 시스템, LLM 기반 RAG 시스템 구축
* 목표: 지식 생산자와 소비자를 연결하는 플랫폼으로 확장하여 새로운 가치와 수익 모델을 창출한다.
* 성공 지표: 플랫폼 총 거래액(GMV), 전문가 콘텐츠 구독자 수, 프리미엄 노트 판매량.

5. 화면별 상세 기능 요구사항 (Screen-by-Screen Functional Requirements)

5.1. 메인(서재) 화면
- [ ] 내 서재에 등록된 모든 책 목록을 상태별(전체/읽는 중/완료/읽고 싶은)로 탭 전환하여 볼 수 있다.
- [ ] 각 책 카드는 표지, 제목, 저자, 상태, 인용문/메모/독서시간 요약 정보를 표시한다.
- [ ] 책 카드를 클릭하면 해당 책의 상세 화면으로 이동한다.
- [ ] 오늘의 독서 기록(총 시간, 페이지, 노트 수, 세션별 기록)을 요약 카드로 제공한다.
- [ ] 상단에서 책 추가, 독서 타이머, 통계 화면으로 바로 이동할 수 있다.

5.2. 책 추가/검색 화면
- [ ] 책 검색 탭에서 키워드로 책을 검색하고, 검색 결과에서 책을 선택해 내 서재에 추가할 수 있다.
- [ ] 검색 결과가 없거나 직접 입력을 원할 경우, 수동으로 책 정보를 입력해 추가할 수 있다.
- [ ] 책 추가 시 표지 색상, 제목, 저자, 상태(읽고 싶은/읽는 중/완료) 등을 입력할 수 있다.

5.3. 책 상세 화면
- [ ] 선택한 책의 표지, 제목, 저자, 상태, 진행률, 인용문/메모/독서시간 요약 정보를 표시한다.
- [ ] 인용문 추가: 직접 입력 또는 OCR(카메라)로 텍스트를 인식해 인용문을 추가할 수 있다.
- [ ] 인용문에는 메모, 태그, 페이지 정보 등을 함께 기록할 수 있다.
- [ ] 감상 메모 추가: 자유롭게 메모를 남기고, 태그를 지정할 수 있다.
- [ ] 등록된 인용문/메모 목록을 확인하고, 태그별로 필터링할 수 있다(필터링은 추후 확장).
- [ ] 책의 독서 상태(읽는 중/완료 등)를 변경할 수 있다.

5.4. 독서 타이머 화면
- [ ] 독서 시작/일시정지/종료 타이머 기능을 제공한다.
- [ ] 타이머 종료 시 읽은 페이지 수, 메모, 인용문 등을 입력할 수 있다.
- [ ] 오늘의 독서 기록에 자동으로 반영된다.

5.5. 통계(대시보드) 화면
- [ ] 연간 목표(읽을 책 수)와 달성률(진행률 바)을 시각적으로 보여준다.
- [ ] 총 독서 시간, 총 페이지, 연속 기록(스트릭), 월별/장르별 차트 등 주요 지표를 제공한다.
- [ ] 최근 읽은 책 목록과 평점을 확인할 수 있다.
- [ ] 통계 데이터를 텍스트로 공유(복사/공유 API)할 수 있다.

5.6. 프로필 화면
- [ ] 사용자 기본 정보(닉네임, 프로필 이미지 등)를 표시한다.
- [ ] (확장) 내 기록/성장 히스토리, 설정 바로가기 등 제공 가능

5.7. 설정 화면
- [ ] 라이트/다크 모드 전환 기능을 제공한다.
- [ ] (확장) 데이터 백업/복원, 계정 관리, 알림 설정 등
